# 登录信息

## 测试账号

系统已自动创建了一个测试管理员账号：

- **用户名**: `admin`
- **密码**: `admin123`

## 访问系统

1. 访问前端系统：http://localhost:8081
2. 使用上述账号登录
3. 登录后可以访问所有功能

## 创建新账号

如需创建新的管理员账号，请在后端目录执行：

```bash
cd backend
source venv/bin/activate
python manage.py createsuperuser
```

按提示输入用户名、邮箱和密码即可。

## 功能说明

### 登录后可以：

✅ 访问工作台仪表盘  
✅ 管理施工单（增删改查）  
✅ 跟踪工序进度  
✅ 管理客户信息  
✅ 管理工序模板  
✅ 管理物料库存  
✅ 查看统计数据  
✅ 访问 Django Admin 后台

### 未登录时：

❌ 自动跳转到登录页面  
❌ API 请求返回 401 错误  
❌ 无法访问任何业务功能  

## 安全建议

⚠️ **生产环境请务必修改默认密码！**

1. 修改默认管理员密码
2. 创建专用的业务账号
3. 根据需要配置不同的权限级别
4. 启用 HTTPS
5. 配置更强的密码策略

## 权限说明

当前系统使用 Django 的 Session 认证：

- ✅ 支持记住登录状态（24小时）
- ✅ 支持多用户同时登录
- ✅ 自动处理登录过期
- ✅ CSRF 保护

## 故障排查

### 无法登录？

1. 检查用户名和密码是否正确
2. 确认后端服务是否正常运行
3. 查看浏览器控制台是否有错误
4. 检查网络连接

### 登录后立即退出？

1. 检查浏览器是否禁用 Cookie
2. 清除浏览器缓存和 Cookie
3. 检查 CORS 配置

### API 返回 401？

1. 可能是登录已过期，请重新登录
2. 检查浏览器 Cookie 是否正常
3. 确认后端 Session 配置正确
