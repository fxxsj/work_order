# Requirements: 印刷施工单任务即时分派系统

**Defined:** 2026-01-30
**Core Value:** 创建即分派，审核即开工 - 施工单一经创建即可预览所有任务，审核通过后任务立即可用

## v1 Requirements

### 任务生成

- [x] **TASK-01**: 施工单创建时立即为所有工序生成草稿任务 ✅ v1.0
- [x] **TASK-02**: 草稿任务可以预览、编辑、删除 ✅ v1.0
- [x] **TASK-03**: 施工单修改工序时提示是否更新已有任务 ✅ v1.0
- [x] **TASK-04**: 草稿任务可以批量操作（批量删除、批量编辑） ✅ v1.0
- [x] **TASK-05**: 施工单审核通过后，所有草稿任务自动转为正式任务 ✅ v1.0
- [x] **TASK-06**: 施工单审核拒绝后，所有草稿任务自动删除 ✅ v1.0

### 任务分派

- [x] **DISP-01**: 系统根据部门优先级规则自动将任务分派到部门 ✅ v1.0
- [x] **DISP-02**: 支持在配置界面设置每个工序的部门优先级 ✅ v1.0
- [x] **DISP-03**: 部门主管可以将任务分配给部门内的操作员 ✅ v1.0
- [x] **DISP-04**: 操作员可以将未分配的任务认领给自己 ✅ v1.0
- [x] **DISP-05**: 一个任务同时只能分配给一个操作员 ✅ v1.0
- [x] **DISP-06**: 支持智能负载平衡（选择当前任务最少的部门） ✅ v1.0

### 任务可见性

- [x] **VIS-01**: 所有部门可以看到所有任务 ✅ v1.0 (implemented in Phase 5)
- [x] **VIS-02**: 支持按部门过滤任务视图 ✅ v1.0 (implemented in Phase 5)
- [x] **VIS-03**: 支持按状态过滤任务（草稿、正式、已分派、进行中、已完成） ✅ v1.0 (implemented in Phase 5)
- [x] **VIS-04**: 支持任务搜索（按任务号、施工单号、内容） ✅ v1.0 (implemented in Phase 5)
- [x] **VIS-05**: 操作员只能看到分配给自己部门的任务 ✅ v1.0 (implemented in Phase 5)

### 施工单详情页集成

- [x] **WO-01**: 在施工单创建/编辑页面显示所有关联任务 ✅ v1.0
- [x] **WO-02**: 支持在施工单页面编辑草稿任务 ✅ v1.0
- [x] **WO-03**: 施工单页面显示任务的分派状态 ✅ v1.0
- [x] **WO-04**: 支持在施工单页面手动添加额外任务 ✅ v1.0
- [x] **WO-05**: 施工单页面显示任务数量统计 ✅ v1.0

### 任务管理页面

- [x] **PAGE-01**: 提供独立页面管理所有任务 ✅ v1.0 (implemented in Phase 5)
- [x] **PAGE-02**: 支持任务列表的分页和虚拟滚动 ✅ v1.0 (implemented in Phase 5)
- [x] **PAGE-03**: 支持批量操作（批量分配、批量删除） ✅ v1.0 (implemented in Phase 5)
- [x] **PAGE-04**: 支持任务高级搜索和筛选 ✅ v1.0 (implemented in Phase 5)
- [x] **PAGE-05**: 支持导出任务列表 ✅ v1.0 (implemented in Phase 5)

### 部门主管界面

- [x] **SUP-01**: 部门主管可以查看本部门所有任务 ✅ v1.0
- [x] **SUP-02**: 支持部门主管将任务拖拽分配给操作员 ✅ v1.0
- [x] **SUP-03**: 显示部门内操作员的工作负载统计 ✅ v1.0
- [x] **SUP-04**: 支持部门主管批量分配任务 ✅ v1.0
- [x] **SUP-05**: 支持部门主管重新分配任务（如果未开始） ✅ v1.0

### 操作员任务中心

- [x] **OP-01**: 操作员可以查看分配给自己的任务 ✅ v1.0
- [x] **OP-02**: 操作员可以查看本部门可认领的任务 ✅ v1.0
- [x] **OP-03**: 支持操作员一键认领任务 ✅ v1.0
- [x] **OP-04**: 显示操作员的个人工作负载统计 ✅ v1.0
- [x] **OP-05**: 支持操作员更新任务进度和完成数量 ✅ v1.0

### 实时通知

- [x] **NOTIF-01**: 任务分配时实时通知相关操作员 ✅ v1.0 (implemented in Phase 8)
- [x] **NOTIF-02**: 任务状态变更时实时通知部门主管 ✅ v1.0 (implemented in Phase 8)
- [x] **NOTIF-03**: 使用 WebSocket 实现实时推送 ✅ v1.0 (implemented in Phase 8)
- [x] **NOTIF-04**: 支持通知历史记录查看 ✅ v1.0 (implemented in Phase 8)

### 优先级配置界面

- [x] **CFG-01**: 提供界面配置工序与部门的关联关系 ✅ v1.0
- [x] **CFG-02**: 支持设置每个工序的部门优先级 ✅ v1.0
- [x] **CFG-03**: 支持启用/禁用分派规则 ✅ v1.0
- [x] **CFG-04**: 支持查看当前分派规则的效果预览 ✅ v1.0

### 性能优化

- [x] **PERF-01**: 批量任务生成使用 bulk_create()（避免 N+1 查询） ✅ v1.0
- [x] **PERF-02**: 添加数据库组合索引优化查询 ✅ v1.0 (implemented in Phase 9)
- [x] **PERF-03**: 前端任务列表使用虚拟滚动（避免渲染卡顿） ✅ v1.0 (implemented in Phase 5)
- [x] **PERF-04**: 使用 Redis 缓存任务统计数据 ✅ v1.0 (implemented in Phase 9)
- [x] **PERF-05**: 任务列表查询使用 select_related/prefetch_related ✅ v1.0 (implemented in Phase 5/9)

### 数据一致性

- [x] **CONS-01**: 使用乐观锁防止并发任务认领冲突 ✅ v1.0
- [x] **CONS-02**: 工序修改后同步更新相关任务 ✅ v1.0
- [x] **CONS-03**: 删除施工单时级联删除所有任务 ✅ v1.0
- [x] **CONS-04**: 草稿-正式流转时验证数据完整性 ✅ v1.0

### 权限控制

- [x] **AUTH-01**: 基于现有 Django 权限系统控制任务访问 ✅ v1.0 (partial, core implemented)
- [x] **AUTH-02**: 操作员只能编辑分配给自己的任务 ✅ v1.0
- [x] **AUTH-03**: 部门主管可以编辑本部门所有任务 ✅ v1.0
- [x] **AUTH-04**: 制表人和业务员可以编辑草稿任务 ✅ v1.0

## Out of Scope

| Feature | Reason |
|---------|--------|
| 复杂调度算法 | v1使用简单负载平衡，复杂调度需要更多业务规则和数据分析 |
| 任务重新分配 | 已开始的任务不允许更换操作员，避免混乱和责任不清 |
| 移动原生应用 | v1仅支持Web，移动端需求待验证后再考虑 |
| 历史数据迁移 | v1仅对新创建的施工单生效，历史数据保持原状 |
| 多租户支持 | 系统目前服务于单一公司，无需多租户复杂度 |
| 离线支持 | 任务分派需要实时数据同步，离线模式优先级低 |
